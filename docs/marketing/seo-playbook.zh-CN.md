# auto-fix-openclaw SEO 整套方案（中文）

## 1. SEO目标

### 核心目标

- 让目标用户在搜索“龙虾断联 / OpenClaw 稳定性 / 自动修复”时找到项目
- 将流量导向三个转化动作：
  1. 访问 GitHub README
  2. 阅读中文项目介绍
  3. 私信/留言获取部署模板

### 目标人群

- 使用 OpenClaw/龙虾的开发者与运维工程师
- 需要跨产品适配和升级稳定性的技术负责人

## 5. 内容生产计划（8周）

每周 2 篇，总计 16 篇，内容类型组合：

- 40% 问题排查类（断联、升级、配置漂移）
- 30% 方案拆解类（探测、修复链、熔断）
- 20% 对比复盘类（人工 vs 自愈）
- 10% 案例与数据类（恢复时间、值守成本变化）

示例题目：

1. 龙虾经常断联的 5 个根因与治理路径
2. OpenClaw 升级后为什么容易“看似正常但实则不稳”
3. 为什么自动修复必须“确定性优先，AI 兜底”
4. 如何用冷却/日上限/熔断防止修复风暴

## 6. 外链与分发策略

### 主阵地

- GitHub 仓库 + README
- Gate 首页静态页

### 分发渠道

- 小红书（问题故事切入）
- 知乎（技术分析长文）
- 掘金/公众号（工程实践复盘）
- 技术社群（私域转化）

### 外链原则

- 每篇内容只放 1 个主入口链接（避免分散）
- 优先引导至 README 或中文介绍页

---

## 7. 技术SEO检查清单

- 页面 Title/Description 唯一且含核心关键词
- H1 唯一且与搜索意图一致
- 图片含 `alt` 文本（已在 Gate 页与 README 处理）
- 页面加载快（SVG + 纯静态）
- 结构化文档内链清晰（README -> 文档矩阵）

---

## 8. 转化指标看板

建议跟踪以下指标（按周）：

- 仓库访问量（unique visitors）
- README 点击到中文介绍页的跳转率
- 小红书内容收藏/私信率
- 模板领取数（评论“模板”）
- 实际部署咨询数
